This walkthrough should cover questions about adding stylesheets to your github.io site. If you haven't added any html files to your repository, back up and follow the main walkthrough.I can view the files in your GitHub repository, so if you get stuck, let me know and I’ll take a look.

You can see examples of stylesheet linking in my own example at https://cdmandrews.github.io/codecademy/. Or look right at my html files by going to https://github.com/cdmandrews and digging around in the codecademy folder of my own cdmandrews.github.io repository.

Remember in the Codecademy lessons for 3, 4, 5, and 6 that you had two pages that you edited for each project. The html pages (which you’ve saved as something like project5.hml and project6.html) and files called stylesheet.html.

On each of those html pages, you have a line that looks like this:

<link type="text/css" rel="stylesheet" href="stylesheet.css"/>

This tells the visitor's browser to load the file stylesheet.css and apply those styles to the html file. If you don't upload the stylesheet.css file, the beautiful html resume page you worked so hard on in Codecademy won't look right.

To make the html pages display with the styles and colors you designed, you’ll need to add the two stylesheets you edited.

Step 1

Use the same process you did to create the html pages, but save them as stylesheet3.css (for the 3rd project), stylesheet4.css (for the 4th project),stylesheet5.css (for the 5th project) and stylesheet6.css (for the 6th). You can name these files however you want; I'm using those names to keep it simple. In this case, you must have separate files for the separate pages (see sidebar).

Why so many stylesheets, man?

In a normal website, you’d have the same stylesheet for all of the pages in the site. That's why all the pages on my cdmandrews.github.io site share the same style. All of the styles are on the linked stylesheet, so I don't have to update styles on all of those individual html pages. That would not too terrible a task for a small site like you're building, but on a big corporate or organizational site, having to change every single html page because someone wanted the backgrounds to be red instead of blue would be a nightmare.

Step 2 (option a)

After you've added the stylesheets to your repository, replace the original <link> line with the following lines of code on these html pages:

On project5.html: 

<link type="text/css" rel="stylesheet" href="stylesheet5.css"/>

On project6.html: 

<link type="text/css" rel="stylesheet" href="stylesheet6.css"/>

(Do the same for 3 and 4)

Step 2 (option b)

If you put your stylesheets in a folder named /css/ rather than the same folder as all of your html pages, you have to add that directory pathway to your href link. This is called "relative linking." Use the following lines of code:

On project5.hmtl: 

<link type="text/css" rel="stylesheet" href="/css/stylesheet5.css"/>

On project6.html: 

<link type="text/css" rel="stylesheet" href="/css/stylesheet6.css"/>

(do the same for 3 and 4)
